# Data Directory Structure

Directorio que contiene todos los resultados de tests, reportes y visualizaciones generadas por el sistema CBR.

## Organización de Directorios

```
data/
├── results/          # Resultados JSON de tests
│   ├── test_adaptive_weights.json
│   ├── test_adaptive_learning.json
│   ├── test_user_simulation.json
│   ├── test_complete_cbr_cycle.json
│   ├── test_semantic_retrieve.json
│   ├── test_semantic_retain.json
│   ├── test_semantic_cultural_adaptation.json
│   ├── test_negative_cases.json
│   ├── master_test_report.json    # Consolidado de todos los tests
│   ├── llm_simulation_results.json           # Simulación LLM (opcional)
│   └── llm_simulation_results_learning.json  # Historial de aprendizaje LLM
│
├── reports/          # Reportes formateados
│   ├── FORMAL_REPORT.md           # Reporte académico principal
│   └── test_summary.csv           # Resumen CSV
│
├── plots/            # Gráficos PNG (8 archivos)
│   ├── feedback_evolution.png
│   ├── feedback_correlation.png
│   ├── weight_evolution.png
│   ├── cultural_retrieval_quality.png
│   ├── cbr_cycle_performance.png
│   ├── adaptation_intensity.png
│   ├── negative_learning.png
│   └── retention_strategies.png
│
└── htmls/            # Reportes HTML interactivos (9 archivos)
    ├── index.html                              # Índice de navegación
    ├── report_adaptive_weights.html
    ├── report_adaptive_learning.html
    ├── report_user_simulation.html
    ├── report_complete_cbr_cycle.html
    ├── report_semantic_retrieve.html
    ├── report_semantic_retain.html
    ├── report_semantic_cultural_adaptation.html
    └── report_negative_cases.html
```

## Categorías de Archivos

### Resultados de Tests (results/)
- **test_adaptive_weights.json** - Evolución de pesos adaptativos
- **test_adaptive_learning.json** - Métricas de aprendizaje
- **test_user_simulation.json** - Simulación multi-usuario
- **test_complete_cbr_cycle.json** - Ciclo CBR completo 4R
- **test_user_simulation.json** - Multi-user simulation with learning
- **test_adaptive_weights.json** - Adaptive weight optimization
- **test_semantic_cultural_adaptation.json** - Cultural adaptation tests
- **test_semantic_retrieve.json** - Semantic similarity in RETRIEVE phase
- **test_negative_cases.json** - Negative case learning
- **test_semantic_retain.json** - Semantic similarity in RETAIN phase
- **test_adaptive_learning.json** - Static vs adaptive comparison
- **master_test_report.json** - Consolidated results from all tests

### Simulation Results (results/)
- **llm_simulation_results.json** - LLM-based evaluations
- **llm_simulation_results_learning.json** - Weight evolution with LLM feedback
- **simulation_static.json** - Static weight simulation (deprecated)
- **simulation_adaptive.json** - Adaptive weight simulation (deprecated)
- **adaptive_evolution*.json** - Weight evolution tracking (deprecated)

### Reports (reports/)
- **FORMAL_REPORT.md** - Academic-style formatted report
- **test_summary.csv** - Metrics summary (Excel/spreadsheet compatible)
- **explicacion_completa_ejemplo.txt** - Example explanations

### Visualizations (plots/)
- **lr_scheduler_comparison.png** - Learning rate scheduler comparison
- Additional plots generated by analysis scripts

## Generating Reports

### Run All Tests
```bash
python run_all_tests.py
```
This executes all formal tests and saves results to `results/`

### Generate Formal Report
```bash
python generate_formal_report.py
```
Creates formatted reports in `reports/`

### Generate Standalone HTML
```bash
python generate_standalone_html.py
```
Creates interactive HTML visualizations in `htmls/`

## File Naming Convention

- `test_*.json` - Results from formal test suite
- `groq_*.json` - LLM simulation data
- `simulation_*.json` - General simulation results
- `*_learning*.json` - Learning/evolution tracking
- `master_test_report.json` - Consolidated results

## Notes

- All timestamps are in ISO 8601 format
- JSON files use UTF-8 encoding
- Test results are overwritten on each run
- Reports are regenerated from `master_test_report.json`
